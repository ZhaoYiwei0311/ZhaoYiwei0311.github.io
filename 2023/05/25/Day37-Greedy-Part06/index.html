738. Monotone Increasing Digits

Intro

An integer has monotone increasing digits if and only if each pair of adjacent digits x and y satisfy x <= y.

Given an integer n, return the largest number that is less than or equal to n with monotone increasing digits.

Solution

1.Greedy

Local optimal is to set previous digit to 9 and decrease current one.

For example, 321 -> 319 -> 299

But numbers like 100 can get confusing for it will result in 90.

The point here is that instead of setting 9 one by one, the better solution would be to set an index that points to the place, after which all digits would be set to 9.

For example, 100. Its index would be 1 since 1 at index 0 is bigger than 0 at index 1. So all digits starting from 1 would be set to 9 as well as decreasing 1 at index 0 to 0.

968. Binary Tree Cameras

Intro

You are given the root of a binary tree. We install cameras on the tree nodes where each camera at a node can monitor its parent, itself, and its immediate children.

Return the minimum number of cameras needed to monitor all nodes of the tree.

Solution

1.Greedy + Postorder

Local optimal is to set camera at the parent node of leaf node, which means we are traversing from bottom to top. Why not from top to bottom is because bottom to top gives an exponential advantage than from root which is 1.

We can set status indicator to nodes. There are 3 statuses. 0 is uncovered, 1 is camera and 2 is covered.

If we find at least one child of current node is uncovered, we set camera to it. This would be the first judgement in each postorder recursion’s logic part so that we will not leave any node uncovered. The second judgement would be to set current node to covered if it has any camera child and we simply update current node’s status to covered and return without doing anything else in this case. These two judgements would cover all situations.
